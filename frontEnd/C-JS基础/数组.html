# 数组的一些小知识

const fruits = ['Apple', 'Banana', 'Orange']

复制一个数组

const fruitsCopy = fruits.slice()

如果访问数组的索引是一个无效值，并不会报错而是会返回 undefined

console.log(fruits[-1]) // undefined
console.log(fruits[10]) // undefined

在JavaScript中，数字开头的属性不能用点号引用,会抛出SyntaxError异常（这在对象里面也一样）

console.log(fruits.0) // SyntaxError

数组中括号里面的索引fruits[1] 也可以写成 fruits['1']，fruits[1]中的1会被 JavaScript 解释器通过调用 toString 隐式转换成字符串

console.log(fruits[1]) // Banana
console.log(fruits['1']) // Banana

比如中括号里面如此设置索引值

console.log(fruits[[1]]) //Banana
console.log(fruits[[[1]]]) // Banana

使用一个合法的下标给数组赋值，当这个下标超出数组大小的时候，会改变数组长度，中间值为empty

fruits[10] = 'Pear'
console.log(fruits) // ['Apple', 'Banana', 'Orange', empty × 7, 'Pear']
console.log(Object.keys(fruits)) // ['0', '1', '2', '10']
console.log(fruits[6]) // undefined

或者显示改变length的值，当length赋一个更大值

fruits.length = 10
console.log(fruits) // ['Apple', 'Banana', 'Orange', empty × 7]
console.log(Object.keys(fruits)) // ['0', '1', '2']
console.log(fruits[6]) // undefined

当赋一个更小值，会删除一些数组元素

fruits.length = 2
console.log(fruits) // ['Apple', 'Banana']
console.log(Object.keys(fruits)) // ['0', '1']

当Array作为函数而不是构造函数使用时，会创建并初始化一个Array对象，它等价于new Array(...)

console.log(Array(1, 2, 3))
console.log(new Array(1, 2, 3))

# 数组静态方法

# Array.from()

对一个伪数组（拥有一个 length 属性和若干索引属性的任意对象）或可迭代对象创建一个新的，浅拷贝的数组。

Array.from() 有一个可选参数mapFn，可以在最后生成的数组上再执行一次 map 方法后再返回。也就是Array.from(obj, mapFn, thisArg) 就相当于 Array.from(obj).map(mapFn, thisArg)

const context = {
	x: '1'
}
console.log(Array.from('foo', function(v, i){ return v+this.x }, context)) // ['f1', 'o1', 'o1']
console.log(Array.from({ length:5 },()=> false )) // [false, false, false, false, false]
const range = (start, stop, step) => Array.from({ length: (stop - start) / step + 1}, (_, i) => start + (i * step));
console.log(range(0, 4, 1)) // [0, 1, 2, 3, 4]

# Array.of()

Array.of() 和 Array 构造函数之间的区别在于处理整数参数：Array.of(7) 创建一个具有单个元素 7 的数组，而 Array(7) 创建一个长度为7的空数组

console.log(Array(7)) // [empty × 7]
console.log(Array.of(7)) // [7]

Polyfill： 假如不存在Array.of()，则在其他代码之前运行下面的代码将创建该方法

if(!Array.of){
	Array.of = function(){
		return Array.prototype.slice.call(arguments)
	}
}

# 数组isArray方法和instanceof的区别

Array.isArray() 用于确定传递的值是否是一个 Array。

console.log(Array.isArray([]))
// 鲜为人知的事实：其实 Array.prototype 也是一个数组
console.log(Array.isArray(Array.prototype))

instanceof 和 isArray

确定一个对象是否是一个数组，这是一种简单的检测方式

if (o instanceof Array){
  //...
}

但是考虑运行在浏览器中的JavaScript时，问题就出现了：可能有多个全局对象。每个window对象都是其页面的全局对象，而o instanceof Array只能在数组是由当前页面的Array创建时才能正常工作。

var iframe = document.createElement('iframe');
document.body.appendChild(iframe);
xArray = window.frames[window.frames.length - 1].Array;
var arr = new xArray(1, 2, 3);

console.log(Array.isArray(arr)) // true
console.log(arr instanceof Array) // false

有没有其他方法可以确定一个值是否是一个数组可以解决这个问题？

另外一种方法依赖于 鸭子类型 ，如果一个值看起来像一个鸭子，那么它就是一只鸭子

这种风格的一个测试是 Object.prototype.toString.call(o) === "[object Array]"，但这依赖于Object.prototype.toString并且 Function.prototype.call没有被改变

由于这些原因，ECMAScript 5 定义了一个方法Array.isArray，来完全解决这个问题

Array.isArray2 = function(arg) {
	return Object.prototype.toString.call(arg) === '[object Array]';
};

Object.prototype.toString = function(){
	return '[object Object]'
}

console.log(Array.isArray2([])) // false
console.log(Array.isArray([])) // true

Polyfill： 假如不存在Array.isArray()，则在其他代码之前运行下面的代码将创建该方法(Tips: 虽然如刚刚所讲，这个方法也不是那么可靠，但在ECMAScript 5之前，也没有什么方法比它更好了，我们只能选择相信它是可靠的)

if (!Array.isArray) {
  Array.isArray = function(arg) {
    return Object.prototype.toString.call(arg) === '[object Array]';
  };
}

# 比较 [@@iterator]() 、values() 、entries()、keys() 的异同

values() 方法返回一个新的 Array Iterator 对象，该对象包含数组每个索引的值

默认情况下，数组的 iterator 方法，与 values() 返回值相同，

// ...
// values: ƒ values()
// Symbol(Symbol.iterator): ƒ values()
// ...

entries() 方法返回一个新的Array Iterator对象，该对象包含数组中每个索引的键/值对

for (let i of fruits.entries()) {
	console.log(i)
}
// (2) [0, 'Apple']
// (2) [1, 'Banana']
// (2) [2, 'Orange']

keys()方法返回一个包含数组中每个索引键的Array Iterator对象。

for (let i of fruits.keys()) {
	console.log(i)
}
// 0
// 1
// 2

# 访问数组项目 Array.prototype.at() 和方括号arr[index] 异同

at()方法拥有更好的简洁性和可读性

console.log(fruits.at(-1))
console.log(fruits.at(fruits.length-1))

# 测试一个数组every()、filter()、some() 的异同

相同点：用指定函数来测试数组中的元素

every() 测试一个数组内的所有元素是否都能通过某个指定函数的测试

所有callback都返回true结果才是true，空数组返回true

const numbers = [1, 6, 16, 24, 45, 88]
console.log(numbers.every((ele) => ele < 100)) // true

some() 方法测试数组中是不是至少有1个元素通过了被提供的函数测试

有一个callback返回true结果就为true，空数组返回false

console.log(numbers.some((ele) => ele > 80)) // true

filter() 方法返回一个新数组, 其包含通过所指定函数的测试的所有元素

console.log(numbers.filter((ele) => ele < 50)) // [1, 6, 16, 24, 45]

# 查找一个数组 find()、findIndex()、indexOf()、includes()、lastIndexOf() 的异同

相同点：返回值或者是下标

find() 方法返回数组中满足提供的测试函数的第一个元素的值

不一定是一个确定的元素，只要元素值满足条件即可

console.log(numbers.find((ele) => ele < 50)) // 1

findIndex() 方法返回数组中满足提供的测试函数的第一个元素的索引。若没有找到对应元素则返回-1

和find不同，它找到的是索引，而不是值

console.log(numbers.findIndex((ele) => ele < 50)) // 0

includes() 方法用来判断一个数组是否包含一个指定的值，根据情况，如果包含则返回 true，否则返回 false

includes() 方法不要求this值是数组对象，所以他可以被用于其他类数组对象

console.log(numbers.includes(24)); // true
(function() {
	console.log([].includes.call(arguments, 2)) // true
})(1, 2, 3)

indexOf() 方法返回在数组中可以找到一个给定元素的第一个索引，如果不存在，则返回-1

和includes相比较，它返回的是下标

console.log(numbers.indexOf(24)); // 3

lastIndexOf() 方法返回指定元素在数组中的最后一个的索引，如果不存在则返回 -1。从数组的后面向前查找，从 fromIndex 处开始

console.log(numbers.lastIndexOf(24)); // 3

# map() 与 parseInt()

来看一段代码

["1", "2", "3"].map(parseInt);

我们期望输出 [1, 2, 3], 而实际结果是 [1, NaN, NaN].

有时候一些简写会让人看起来很迷茫，不妨换一种写法

["1", "2", "3"].map((v, i) => {
  return parseInt(v, i)
})

这样看起来就清楚了,再来看一看 parseInt 的语法

parseInt(string, radix)

当参数 radix 的值为 0，或没有设置该参数时，parseInt() 会根据 string 来判断数字的基数。

当忽略参数 radix , JavaScript 默认数字的基数如下:

  • 如果 string 以 "0x" 开头，parseInt() 会把 string 的其余部分解析为十六进制的整数。
  • 如果 string 以 0 开头，那么 ECMAScript v3 允许 parseInt() 的一个实现把其后的字符解析为八进制或十六进制的数字。
  • 如果 string 以 1 ~ 9 的数字开头，parseInt() 将把它解析为十进制的整数。

回头再分析下迭代步骤

第一次循环 parseInt('1',0)，返回 1
第一次循环 parseInt('2',1)，返回 NAN，没有 1 进制
第一次循环 parseInt('3',2)，返回 NAN，2进制不存在3

# Array.prototype.toString

Array对象覆盖了Object的 toString 方法。对于数组对象，toString 方法连接数组并返回一个字符串，其中包含用逗号分隔的每个数组元素。

当一个数组被作为文本值或者进行字符串连接操作时，将会自动调用其 toString 方法。
