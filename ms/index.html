# px % em rem vw/vh 区别

  • px是基本单位**（相对长度单位，相对于屏幕）**，其他是相对单位

  • %相对于父元素的宽度比例

  • em 在font-size中使用相对于父元素的字体大小，在其他属性中使用相当于自身的字体大小，如width

  • rem 相对于根节点html的font-size

  • vw 屏幕宽度的1%

  • vh 屏幕高度的1%

  • vmin 两者最小值

  • vmax 两者最大值

# for in 和 for of

  • for in ，可枚举的数据，对象、字符串、数组，得到key

  • for of， 可迭代的数据，数组、字符串、Map、Set ，得到value

  • for await of 迭代一个promise数组，相当于promise.all()

# scrollHeight、offsetHeight和clientHeight的区别

  • scrollHeight：padding + 实际内容尺寸（overflow:scroll）
  • offsetHeight：boder + padding + content
  • clientHeight：padding + content

# WebSocket和Http协议的区别

  1. WebSocket的协议名是ws://，可双端发起请求
  2. WebSocket协议没有跨域限制
  3. 通过send和onMessage通讯，双端都一样(http协议通过req，res)

# WebSocket和Http长轮询的区别

  • WebSocket双端通讯，有消息就发给客户端
  • Http长轮询不断发送请求，直到有结果

# 性能优化切入点

  • 请求： 减少http请求，例如雪碧图、内联base64图、合并JS和css表
  • **CDN：**内容发布网络（CDN）：因为缓存和各地访问各地的CDN服务器（类似于京东每个城市的仓库）
  • **压缩：**gzip压缩
  • **渲染顺序：**样式放顶部，脚本放底部
  • **缓存：**Expires头（设置的是一个日期，需要服务器时间同步，并且还要动态的修改日期）和Cache-Control（优先级更高，http1.1引入 Cache-Control:max-age=132332234,第一次请求到现在的时间）缓存文件

# Vue2.0和3.0响应式的区别

# Vue2.0

Vue2.0中使用ES5中的Object.defineProperty方法实现响应式数据 缺点 无法监测到对象属性的动态添加和删除 无法监测到数组的下标和length属性的变更 解决方案 Vue2.0提供Vue.set方法用于动态给对象添加属性 Vue2.0提供Vue.delete方法用于动态删除对象的属性 重写vue中数组的方法，用于监测数组的变更

# Vue3.0

通过Proxy(代理): 拦截对data任意属性的任意(13种)操作, 包括属性值的读写, 属性的添加, 属性的删除等... 通过 Reflect(反射): 动态对被代理对象的相应属性进行特定的操作 Vue3.0中使用ES6中的proxy语法实现响应式数据 优点 可以检测到代理对象属性的动态添加和删除 可以监测到数组的下标和length属性的变更 缺点 ES6的proxy语法对于低版本浏览器不支持，IE11 Vue3.0会针对于IE11出一个特殊的版本用于支持ie11

# cookie和token区别

  • cookie和session配合使用，有跨域的限制

  • token和Jwt使用，无跨域限制

  • 如果有严格管理用户的需求，比如封禁用户，使用session

  • 如果没有其他特殊要求，推荐使用token
