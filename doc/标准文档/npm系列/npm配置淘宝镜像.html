# 什么是淘宝镜像？

npm 服务器是国外的，有些资源在国内访问很慢，或者是访问不了的。淘宝帮我们把外网的资源复制了一份，放在它的服务器上，以便我们能够高速下载，这就是淘宝镜像 (opens new window)

# 使用（2022-10-25更新）

# tips：原淘宝 npm 域名即将停止解析 (opens new window)

正如在《淘宝 NPM 镜像站喊你切换新域名啦》 (opens new window) 中预告的那样：

http://npm.taobao.org (opens new window) 和 http://registry.npm.taobao.org (opens new window) 将在 2022.06.30 号正式下线和停止 DNS 解析。

域名切换规则：

  • http://npm.taobao.org (opens new window) => http://npmmirror.com (opens new window)
  • http://registry.npm.taobao.org (opens new window) => http://registry.npmmirror.com (opens new window)

因此，有强依赖老域名的各位开发者请尽快更新到新域名

# 设置 npm 代理为淘宝镜像

npm config set registry https://registry.npmmirror.com

# 查看当前 npm 的镜像地址

npm config get registry

# 还原成国外的镜像

npm config set registry https://registry.npmjs.org

# 单次使用

npm --registry https://registry.npmmirror.com install express

# 设置 sass 镜像

npm config set sass_binary_site https://npmmirror.com/mirrors/node-sass/

# 设置electron镜像

npm config set electron_mirror https://npmmirror.com/mirrors/electron/

# 安装淘宝定制的 cnpm (opens new window) 命令行工具代替默认 npm

npm install -g cnpm --registry=https://registry.npmmirror.com

# 使用

# 设置 npm 代理为淘宝镜像

npm config set registry https://registry.npm.taobao.org

# 查看当前 npm 的镜像地址

npm config get registry

# 还原成国外的镜像

npm config set registry https://registry.npmjs.org

# 单次使用

npm --registry https://registry.npm.taobao.org install express

# 设置 sass 镜像

npm config set sass_binary_site https://npm.taobao.org/mirrors/node-sass/

# 设置electron镜像

npm config set electron_mirror https://npm.taobao.org/mirrors/electron/

# 安装淘宝定制的 cnpm (opens new window) 命令行工具代替默认 npm

npm install -g cnpm --registry=https://registry.npm.taobao.org

注意：

  1. 使用 cnpm install 的时候，并不会生成package-lock.json文件。

  2. cnpm install的时候，就算你项目中有package-lock.json文件，cnpm也不会识别，仍会根据package.json来安装。所以这就是为什么之前你用npm 安装产生了package-lock.json，后面的人用cnpm来安装，可能会跟你安装的依赖包不一致。

  3. 这是因为cnpm 不受package-lock.json影响，只会根据package.json进行下载。

因此，不建议使用 cnpm install 来安装项目依赖包。而是使用 npm 代理来解决。
