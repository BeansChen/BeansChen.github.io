为什么要用void 0 代替undefined？

2018-07-04
  • javascript

这几天在框架源码的时候，偶然看到把变量声明为为void 0这种写法，前辈的代码果然是高深莫测，为什么要这样子写呢？这成功的激发了本宝宝的好奇心。


翻阅了多篇文章之后，总算明白了void 0使用来代替undefined，那么为什么直接用undefined不行，非得找个替代品？这个可以追溯到long long ago！

为什么不直接使用undefined呢

原来，undefined并不是保留字，他只是一个全局属性，在低版本的IE中可以被重写

var undefined = 8;
alert(undefined);
// undefined -- IE 10
// 10 -- IE 8

在es5之前在window对象下的undefined都可以被重写，es5之后虽然规定了全局变量的undefined为只读，但是在局部依然可以被改变

function a(){
var undefined = 10;
alert(undefined);//10            
}
a();

为什么用void 0来代替undefined呢

void操作符可以执行一个表达式，并会在在任何情况下总是返undefined，也就是说可以用来完美的替代undefined属性

为什么是void 0 而不是void 1呢？其实这两者并没有什么区别，还有就是使用void 0 还可以减少3个字节。

总结

可以防止undefined被重写，并可以节约3个字节。老司机们真的666啊！
