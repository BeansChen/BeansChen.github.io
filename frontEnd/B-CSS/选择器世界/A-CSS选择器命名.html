# 增加CSS优先级的小技巧

实际开发中，肯定有需要增加CSS优先级的场景，很多人做法是嵌套或者增加标签选择器，但都不是好的办法，可以像下面这么做：

.father .foo{}   嵌套，增加了多余的标签
div.foo{}        加标签选择器，失去灵活性，标签变化css即失效
.foo[class]      优秀
.foo.foo         优秀
#foo[id]         优秀

# CSS选择器是否区分大小写？

  1. 答案是有些区分，有些不区分。
  2. 在html中，标签和属性名是不区分大小写的，相应的在css中也一样。
  3. 但是属性的值是区分大小写的。
<div class="content">我是一个div</div>	

div { padding: 10px;background-color: black;}
[CLASS]{color: #fff;}
.CONTENT{text-decoration: underline;}

例如上面这个元素的class属性就没有生效。

# 一些思考

  • 长命令还是短命名？
  • 单命名还是组合命名？
  • 面向属性的命名和面向语义的命名？
.desc{}
.description{}

.header{}
.app-header{}
.bn-banner{}

.db{display:block;}
.fl{float:left;}
.banner{/*面向语义*/}
  1. 面向语义命名的优点在于复用性强，但是代码笨重，开发效率一般，比如只是给p标签加一个margin-top:10px 也需要创建一个类。
  2. 面向属性命名优点在于复用性强，方便快捷，但是大量使用起来维护起来很困难。

经验总结：

我们把网站归为几块：公共结构、公共模块、UI组件、精致布局（指需要用到很多的css样式）和一些细枝末节。

其中前三种属于页面公用内容，就不适合用面向属性的命名方式，因为这些内容散布在项目的各个角落，一旦需要修改，就得找到所有地方，显然这种维护成本极高。

而精致布局也不适合，因为面向属性的类目属性单一，无法驾驭精致的样式，势必还是需要额外的语义化类目，所以也就大可不必了。

对于细枝末节的需要微调的场景非常适合使用面向属性的命名，它可以解决最后一公里的效率问题。

# 最佳实践

# 1.命名书写

  1. 命名建议用小写，使用英文单词或者缩写，除了特殊的词汇（比如taobao、weibo）不要用拼音，不建议使用驼峰命名，驼峰命名一般给JavaScript DOM使用。
  2. 对于组合命名，可以使用短横线和下划线，也可以使用连续的短横线和下划线，组合数尽量保持在3个以内，最多5个。
  3. 使用统一的前缀，强化品牌的同时也避免样式冲突，例如.nh-header-tit （内部项目，无须外部引入可以不要前缀，否则还是建议带上，避免冲突）
  4. 不要使用ID选择器，ID选择器更多的是给JavaScript使用
  5. 少点嵌套，尤其是less，scss后，css的层级变得很糟糕。很多有照着html的层级去书写css，它是及其脆弱的，一旦html层级变了或者是标签变了，将不得不去调整css样式，而且过深的层级势必造成优先级混乱 。

# 2.选择器使用

根据选择器的使用类型，可以将网站分为3个部分，分别是css重置样式，css基础样式，css交互变化样式

最佳实践：

  • 无标签，无层级
  • 状态类名标识符
  • 面向属性命名的CSS样式库
