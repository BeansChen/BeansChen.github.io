    注意：
    1. 正则表达式，就是一个匹配模式
    2. 正则表达式，本身就是一个字符串（其中有一些特殊字符的语法规则）
    3. 正则的字符串，一定要在函数中使用

      1.基本格式

      定界符号，很多符号都可以，常用/ /

    原子：最小的一个匹配单位

    元字符：不能单独使用，修饰原子，用来扩展或者限定原子的功能

    模式修正符：写在右边

    2.正则中的原子

    2.1 打印字符和非打印字符

    打印字符指键盘可以打出来的字符，非打印字符指例如回车等打印不出来但是存在的字符

    2.2 转义字符
    1. 可以将有意义的字符转为没有意义的字符
    2. 可以将没有意义的字符转为有意义的字符

    3. 没有意义的字符，加上转义也没有意义，可转义也可以不转，不影响

      2.3 特殊的原子

      |code|text|
      |:-:|:-|
      |\d|任意一个数字|
      |\D|任意一个非数字|
      |\w|任意一个字 a-z A-Z 0-9 _|
      |\W|任意一个非字|
      |\s|代表空白 空格，回车，tab键|
      |\S|非空白|
      |.|点代表所有|

      2.4 自定义原子表

      [13579]
      [^a-zA-Z] - 表示从哪到哪，^ 除了…..

      3.正则的元字符

      符号|功能
      :–:|–
      *|匹配 0 次或者多次
      +|匹配 1 次或者多次
      ?|匹配 0 次或 1 次
      |匹配两个或多次分支选择，但优先级是最低的
      {n}|匹配恰好出现n次
      {n,}|匹配 n 次以上
      {n,m}|匹配 n 次到 m 次
      ^ 和 \A|表示必须匹配字符串以什么开始，且必须写在正则表达式最前面
      $ 和 \Z |和上面相反

    5. 正则函数 （功能强，但是效率低）
