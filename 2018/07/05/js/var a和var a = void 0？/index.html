var a和var a = void 0?

2018-07-05
  • javascript

昨天知道了为什么要用void 0 来替代undefined。

1.防止undefined被重写 2.减少3个字节

但直接声明var a;，那么这个变量就是undefined

var a;
console.log(a)；//undefined

那为什么要多此一举多写几个字符呢？

哒哒哒，哒哒哒，我又开始百度啦！然而这次百度并没有给我什么满意的答案。我又看了下大佬的代码，比如这样

function (){
    var a = void 0;
    a = document.getElementById('demo');
}

就这点内容为什么我不直接这样写呢

function (){
    var a = document.getElementById('demo');
}

到这里，我认为在代码还不多的情况下，直接把变量声明在顶部或许更好，那么什么时候用void 0 呢？

var a = 10;
...
var a;
console.log(a);//10

比如这种时候，我们不确定变量是否在哪里声明过，这个时候就可以声明void 0了，就这样吧，巴拉巴拉
