# 背景

在使用 vue-cli 开发项目的时候，发现项目根目录下有.browserslistrc 文件，里面的配置如下

> 1%
last 2 versions
not dead

后面在阅读 @babel/preset-env (opens new window) 文档时候，也有提到：

对于基于浏览器或 Electron 的项目，推荐使用 .browserslistrc 文件来指定目标环境。默认情况下@babel/preset-env将使用browserslist 配置源 (opens new window) ，除非设置了targets (opens new window)或ignoreBrowserslistConfig (opens new window)选项。

所以，在此结合官方文档 (opens new window)对 browserslist 进行简单的介绍。

# 简介

browserslist 是在不同的前端开发工具之间共享目标浏览器和 node 版本的配置工具。它被以下工具使用：

  • Autoprefixer (opens new window)
  • Babel (opens new window)
  • postcss-preset-env (opens new window)
  • eslint-plugin-compat (opens new window)
  • stylelint-no-unsupported-browser-features (opens new window)
  • postcss-normalize (opens new window)
  • obsolete-webpack-plugin (opens new window)

所有的工具都将自动的找到目标浏览器，前提是你在根目录的 package.json 中增加了以下配置：

  "browserslist": [
    "defaults",
    "not IE 11",
    "maintained node versions"
  ]

或者是在根目录新建 .browserslistrc 文件（配置如下）：

# Browsers that we support

defaults
not IE 11
maintained node versions

开发者使用类似 last 2 versions 这样的查询来设置他们的版本列表，避免频繁的手动更新版本。browserslist 使用 Can I Use (opens new window) 网站的数据来查询浏览器版本范围。

Browserslist Example (opens new window) 这个仓库显示了Browserslist如何被各个工具使用。

# 最佳实践

  • 有一个 defaults 查询，他为大多数的用户提供了合理的配置：
  "browserslist": [
    "defaults"
  ]
  • 如果您想更改默认浏览器集，我们建议将 last 2 versions,not dead与> 0.2%. 这是因为last n versions它本身不会添加流行的旧版本，而仅使用上述百分比0.2%从长远来看会使流行的浏览器更加流行。我们可能会遇到垄断和停滞的情况，就像我们在 Internet Explorer 6 中遇到的那样。请谨慎使用此设置。
  • last 2 Chrome versions仅当您使用一个浏览器为信息亭制作 Web 应用程序时，才直接选择浏览器 ( )。市场上有很多浏览器。如果您正在制作通用网络应用程序，您应该尊重浏览器的多样性。
  • 不要因为您不了解浏览器而删除它们。Opera Mini 在非洲拥有 1 亿用户，在全球市场上比 Microsoft Edge 更受欢迎。中国 QQ 浏览器的市场份额超过了 Firefox 和桌面版 Safari 的总和。

# 总结

browserslist 是一个配置文件，主流的前端开发工具都遵循其规则来兼容到目标浏览器。
