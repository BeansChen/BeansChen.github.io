# :link

  • 匹配有href属性+值但没有访问过的<a>元素
  • 但是如果设置了a:link，那么就得设置a:visited，不然访问过后a:link设置的样式就失效了
  • 而且:visted这个伪类，尽量不要使用了，暴露了用户的隐私，哈哈哈~
  • 链接经常还会加上:hover伪类，大家都是伪类，平起平坐，如果把:link放在后面，就会使:hover失效
  • “love-hate 顺序”，:link->:visited->:hover->:active
  • 实际开发设置 a{ }，就好了，没必要这么麻烦
  • :link好处就是可以区分真假链接，但是真有这种场景使用属性选择器[href]{ }似乎是个更好的选择，更加容易理解，不容易造成混乱

# :visited

  • 支持的属性有限
  • 不支持半透明色
  • 只能重置，意思是要设置背景色，只有原本有背景色才行
