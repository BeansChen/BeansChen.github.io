    前言

    为什么要写博客

    以前写笔记都是写在自己的电脑上，但有时候工作遇到了一些笔记上记载了的问题，每次又要重新找答案或是回家再翻笔记，所以也就越来越连笔记也懒得写了。

    把笔记写在博客中，以后遇到找答案方便，如果自己的某篇博客能帮助其他人解惑，那当然更加好了，这是我写博客的初衷。

    为什么选择github pages

    最初有在博客园写过博客，但慢慢的觉得那界面风格有些不能接受，所以就有了搭建个人博客的打算，其实刚开始是想自己设计、自己写后台、买空间、买域名…..。于是乎百度百度，然后百度出了githubpages + hexo 搭建博客，仔细一看，哎这个不错免费空间，而且还有很多风格的模板可以选择，关键的是以上的步骤全省了，使用githubpages有以下优点：

    • 1G以下免费空间，完全够用了，超过似乎会收到GITHUB的通知邮件
    • 需要Git来进行版本管理，对程序员来说熟悉Git是很有必要的
    • 风格可以自定义，功能也可以自定义

    第一步、准备

    使用github搭建博客之前首先要做的准备工作：注册github账号、安装nodejs，安装Git，安装hexo

    注册github账号

    如果还没有github账号，就先注册一个吧 https://github.com/

    注册成功后，打开注册邮箱验证一下就可以了，下面开始新建一个项目

    输入仓库名，与用户名一致，点击新增仓库

    进入仓库，选setting

    点击按钮

    随便选择一个主题，然后在浏览器输入刚刚的仓库名字，比如： https://testsblog.github.io 就可以访问了

    下载安装Git

    首先安装Git并配置好环境变量，随便找一个教程安装就行了，安装Git

    添加SSH Key到github

    检查ssh是否存在：

    打开git Bash：输入 cd ~/.ssh

    如果没有提示，则存在ssh，在系统User/当前用户/.ssh目录下找到id_rsa.pub文件，复制里面的内容，然后

    配置账户
    $ git config --global user.name “用户名” 
$ git config --global user.email “邮箱”
    测试是否成功
    $ ssh -T git@github.com

    如果提示Hi testsblog! You’ve successfully authenticated, but GitHub does not provide shell access.则是成功了

    安装nodejs

    安装nodejs并配置环境变量，随便找个教程就行

    第二步、安装hexo

    新建一个文件夹hexo，打开cmd，进入文件夹，在命令行中依次执行以下步骤：

    1.全局安装hexo脚手架

    npm install -g hexo-cli

    2.初始化hexo

    hexo init hexo

    3.进入文件夹

    cd  hexo

    4.安装依赖

    npm install

    5.部署

    hexo generate

    6.本地测试

    hexo server

    7.在浏览器输入http://localhost:4000既可以访问了

    E:\hexo>hexo server
INFO  Start processing
INFO  Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

    第三步、将本地项目同步到github

    安装hexo-deployer-git插件
    npm install hexo-deployer-git --save
    编辑器打开全局配置文件_config.yml，编辑
    deploy:
  type: git
  repository: ssh://git@github.com/github用户名/github用户名.github.io.git
  branch: master
    部署

    清除缓存

    hexo clean  #清除缓存 网页正常情况下可以忽略此条命令

    生成器，生成静态文件

    hexo g

    推送到github

    hexo d

    最后一行会打印表示推送成功，浏览打开项目testsblog.github.io会发现已经成功

    INFO  Deploy done: git

    第四步，更改主题和其他

    更改主题

    如果想要更改主题，可以到hexo官网找自己喜欢的主题并下载下来https://hexo.io/themes/

    • 放到项目目录下的themes/目录中

    • 更改全局配置文件_config.yml中的theme名字

    常用命令
    hexo new"postName" #新建文章
hexo new page"pageName" #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #将.deploy目录部署到GitHub
hexo help # 查看帮助
hexo version #查看Hexo的版本
