angularJS服务

angular的注入器创建两类对象，服务和专用对象，专用对象包括控制器，指令，过滤器和动画

一个好的angular应用，不该把业务代码放在控制器层，这个时候需要用到angular自定义服务。
angular实现服务的方式有5种：

  • provider
  • factory
  • service
  • constant
  • value

value

用来定义一个值共享的字符串，可以在多个地方使用,不能被注入到config中

StarterCore.value("version",'1.10');

constant

用来定义一个常量，可以被注入到config中

myApp.constant('apiURL', 'http://192.169.0.1:8080');

Service

返回的是一个实例化后的构造函数

app.service('MyService', function () {
    this.sayHello = function () {
        console.log('hello');
    };
});

factory

返回的是一个对象

app.factory('MyFactory', function () {
    return {
        sayHello:function(){
            console.log('hello');
        }
    }
});

provider

可以在注入之前,在config中进行配置

Starter.provider("myProvider",function(){
    var name="bean";
    this.setName = function(n){
        name = n;
    }
    this.$get = function(){
        return {
            name:name,
            height:"175cm"
        }
    }
})
Starter.config(function(myProviderProvider){
    myProviderProvider.setName="dou"
})

总结

  • 注入器使用图纸创建两类对象：服务和专用对象。
  • 总共有5类图纸来定义如何创建对象：Value，Factory，Service，Provicer以及Constant。
  • Factory和Service是最常用的图纸。它们之间的唯一区别就是Service图纸在创建自定义对象时更适用，而Factory还可以创建Javascript原始类型以及函数。
  • Provider图纸是最核心的图纸类型，而其它所有图纸都只是基于它的语法糖。
  • Provider也是最复杂的图纸类型，除非你正在构建需要全局配置的可复用代码，否则不要使用它。
  • 除了控制器，其他所有专用对象都是通过Factory图纸来定义的。
